PURPOSE
-------
This file records cryptographic fingerprints (SHA-256) of the
canonical artifacts of the Bounded Power Framework. Any modification
to a hashed file produces a different hash, making silent alteration
detectable.

The hashes secure each file's integrity within this repository. They
are not enforcement primitives.

The Structural Constitution (SCBP-04) is the only document whose
clause text carries binding authority within the construction. The
operational documents, the registry charter, and the registration
form are hashed because each of them is a canonical artifact
referenced by the Constitution and by the registry's operating logic;
silent alteration of any of these would change what the framework
substantively requires of registrants without changing the
Constitution.

To verify any file: ensure UTF-8 encoding with LF line endings and no
BOM, then run sha256sum on the file from the repository root and
compare against the hash recorded below.

VERIFICATION INSTRUCTIONS
--------------------------
1. File encoding must be UTF-8, no BOM
2. Line endings must be LF (Unix-style, not Windows CRLF)
3. No trailing spaces on any line
4. File must end with exactly one newline character
5. From the repository root, run:
   sha256sum <path-as-listed-above>
6. Compare output to the hash recorded above — must match exactly

If a file's hash does not match its registry entry, the file in this
repository has been altered from the version originally hashed.

This registry itself is not hashed (it would have to hash itself).
The registry's integrity rests on the visibility of the version-
control history of this repository.

================================================

AMENDMENT
---------
Updates to any hashed artifact produce a new entry in this registry.
Prior entries are preserved as historical record. Silent replacement
of an entry without preservation of the prior entry is itself a
finding under R56 (Canonical Integrity Requirement).
