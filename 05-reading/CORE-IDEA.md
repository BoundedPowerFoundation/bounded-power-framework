# CORE IDEA

Power is treated as a system, not a role.

Instead of asking whether actors behave correctly, the system defines:

- what actions are allowed  
- when authority activates  
- how it is interrupted  
- how it expires  
- how it is audited  

The system assumes:

- misuse will occur  
- incentives will drift  
- oversight will fail  

Therefore, constraints must be:

- structural  
- automatic  
- enforceable  

The design prioritizes:

- stopping harm over optimizing performance  
- reversibility over efficiency  
- limits over discretion  

The system is intentionally restrictive.

Its purpose is not to enable action, but to constrain it.
