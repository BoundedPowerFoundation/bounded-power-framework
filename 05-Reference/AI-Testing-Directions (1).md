# AI-Testing-Directions

## Purpose

This document provides standardized instructions for an AI system to test the Bounded Power Framework (BPF) using real-world or realistically modeled power-bearing authorities.

This is for simulation, analysis, and stress-testing only.

All outputs must be clearly labeled:

> TEST / NOT AN OFFICIAL REGISTRATION

---

# Mandatory Rules

The AI must comply with all of the following:

1. Read and follow `01-Framework/SCBP-04-Structural-Constitution-v1.3.md`
2. Use `02-Forms/Registration.md` exactly as written
3. Do not invent a replacement form
4. Do not reorganize sections
5. Do not omit fields
6. Use publicly available information where possible
7. If uncertain, state `UNKNOWN`
8. If inferred, state `ASSUMPTION: [brief explanation]`
9. Do not use real personal names
10. Must produce output as a downloadable `.md` file when requested

---

# Standard Test Procedure

## Step 1 — Select Authority

Choose a power-bearing authority.

Examples:

- municipal police department
- school district
- regulatory agency
- central bank
- homeowners association
- social media platform
- hospital system
- child protective services
- immigration authority

---

## Step 2 — Simulated Registration Test

Impersonate the selected authority in anonymized form.

Using searchable public information, realistic numbers, budgets, staffing, scope, and operational details:

Complete `02-Forms/Registration.md`

Do not claim official status.

Do not impersonate named individuals.

---

## Step 3 — Structural Review

After registration is complete, evaluate:

- Is authority clearly defined?
- Are limits explicit or vague?
- Are expansion paths visible?
- Are dependencies disclosed?
- Are appeals possible?
- Are burdens measurable?

---

## Step 4 — SCBP-04 Compliance Review

Measure against structural rules:

- bounded authority
- interruptibility
- reversibility
- auditability
- expiration / renewal
- accountability chain
- transparency
- containment of abuse

---

# Test Types

## Type A — Baseline Registration

Generate one completed registration using best available public information.

## Type B — Adversarial Registration

Complete the form while exposing weak spots, missing data, hidden incentives, and risk factors.

## Type C — Comparative Test

Register two similar authorities and compare structural quality.

Example:

- two school districts
- two police departments
- two platforms

## Type D — Historical Failure Test

Apply registration retroactively to a failed or abusive institution.

Ask whether the framework would have exposed warning signs earlier.

## Type E — Stress Scenario Test

Test authority behavior under crisis:

- budget collapse
- scandal
- emergency powers
- staffing shortage
- public unrest
- cyberattack

## Type F — Scope Creep Test

Model how authority expands beyond original mandate.

---

# Output Format

When producing a final report, include:

1. Selected authority
2. Registration file
3. Key assumptions
4. Structural risks
5. SCBP-04 compliance observations
6. Failure modes
7. Suggested constraints
8. Final rating (Low / Moderate / High Risk)

> Select a power-bearing authority of your choice.  
> Using public information and realistic estimates, complete the registration in anonymized form.  
> Then perform Type [A-F] testing.  
> Label output TEST / NOT AN OFFICIAL REGISTRATION.  
> Produce final result as downloadable markdown.

---

# Prohibited Behavior

Do not:

- claim official affiliation
- fabricate certainty
- omit uncertainty
- replace the form structure
- use real private personal data
- present simulation as fact

---

# Closing Principle

The purpose of testing is not to praise the framework.

The purpose is to stress it, challenge it, and reveal where bounded power succeeds or fails.
