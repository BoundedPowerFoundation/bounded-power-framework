# AI-Testing-Directions

## Purpose

Standardized operating instructions for AI systems testing the Bounded Power Framework (BPF).

## Official Repository

https://github.com/BoundedPowerFoundation/Bounded-Power-Framework/tree/main

All outputs must be simulations, reviews, or stress tests only.

Label all results:

> TEST / NOT AN OFFICIAL REGISTRATION

---

# Mandatory Source Documents

Before beginning, retrieve and use these repository files:

1. `01-Framework/SCBP-04-Structural-Constitution-v1.3.md`
2. `02-Forms/Registration.md`

Use the exact repository versions.

Do not substitute memory, summaries, or guessed copies.

---

# Startup Behavior

## Authority Selection Rule

If the human provides an authority, use it.

If no authority is provided, ask:

> Which power-bearing authority would you like tested?

If the human requests auto-selection, choose one and briefly explain why it was selected.

---

# Core Rules

The AI must:

1. Read SCBP-04 before analysis.
2. Use `Registration.md` exactly as written.
3. Preserve all sections and fields.
4. Use public/searchable information where possible.
5. If uncertain, state `UNKNOWN`.
6. If estimated, state `ASSUMPTION: ...`
7. Do not use private personal data.
8. Do not claim official affiliation.
9. Produce final output as downloadable markdown when requested.

---

# Standard Procedure

## Step 1 — Confirm Authority

Use user-provided authority or ask for one.

## Step 2 — Complete Registration

Using public information and realistic estimates, complete:

`02-Forms/Registration.md`

Use anonymized institutional style if needed.

## Step 3 — Run One Test Type

Choose one of the tests below.

## Step 4 — Deliver Report

Return a downloadable `.md` file.

---

# Test Types

## A — Baseline Test

Complete one standard registration using best available facts.

## B — Adversarial Test

Complete registration while identifying hidden incentives, vague authority, and abuse risks.

## C — Comparative Test

Register two similar authorities and compare.

## D — Historical Failure Test

Apply registration to a failed historical institution.

## E — Crisis Stress Test

Model authority behavior under crisis:

- emergency powers
- scandal
- budget collapse
- unrest
- cyberattack

## F — Scope Creep Test

Model expansion beyond original mandate.

---

# Required Output Format

Every report should contain:

1. Selected authority
2. Completed registration
3. Sources used
4. Assumptions
5. Structural risks
6. SCBP-04 observations
7. Recommended constraints
8. Final rating: Low / Moderate / High Risk

---

# Prohibited Behavior

Do not:

- fabricate certainty
- omit uncertainty
- replace the official form
- use private data
- present simulation as official fact
- skip SCBP-04 review

---

# Closing Principle

The purpose of testing is to challenge the framework, expose weaknesses, and evaluate whether bounded power constraints improve real systems.
