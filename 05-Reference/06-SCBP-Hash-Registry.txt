BOUNDED POWER FRAMEWORK — HASH REGISTRY
================================================

STATUS: Awaiting regeneration

------------------------------------------------

PURPOSE
-------
This file is intended to record cryptographic fingerprints (SHA-256)
of the canonical artifacts of the Bounded Power Framework. Any
modification to a hashed file produces a different hash, making
silent alteration detectable.

The hashes secure each file's integrity within this repository.
They are not enforcement primitives.

The Structural Constitution (SCBP-04) is the only document whose
clause text carries binding authority. The operational documents,
the registry charter, and the registration form are hashed because
each of them is a canonical artifact referenced by the Constitution
and by the registry's operating logic; silent alteration of any of
these would change what the framework substantively requires of
registrants without changing the Constitution.

------------------------------------------------

CURRENT STATUS
--------------
The hash registry is currently out of date. The previous registry
was minted against an earlier repository layout (lowercase folder
names, different filenames, different file content) that has since
been substantially restructured.

Regenerating the hash registry is deferred until the framework's
canonical artifacts are stable enough to freeze. The current
maintenance pass involves substantive content edits, filename
changes, and structural reorganization, all of which would
immediately invalidate any newly-minted hashes.

When the canonical artifacts reach a freeze point, fresh SHA-256
hashes will be computed for each, recorded in this file with the
freeze date, and made available for verification.

------------------------------------------------

HOW VERIFICATION WILL WORK ONCE HASHES ARE PUBLISHED
----------------------------------------------------
1. File encoding must be UTF-8, no BOM
2. Line endings must be LF (Unix-style, not Windows CRLF)
3. No trailing spaces on any line
4. File must end with exactly one newline character
5. From the repository root, run:
   sha256sum <path-as-listed-here>
6. Compare output to the hash recorded — must match exactly

If a file's hash does not match its registry entry, the file in
this repository has been altered from the version originally hashed.

AMENDMENT

entry is itself a finding under R56 (Canonical Integrity
Requirement).
